# LogicSLM — a critical-reasoning small language model

A **Qwen3.5-4B** LoRA fine-tune that reasons briefly from premises and commits to exactly one
answer — an entailment label (`True / False / Unknown`) or an MCQ option — **including recognizing
when a conclusion does not follow.** Built to make language models more reliable at the logical work
students do: argumentative essays (history, language arts) and debate.

## Results

Held-out accuracy across six logical-reasoning families (n=60/family, greedy decode). Frontier models
scored on the identical items. Parentheses = **neutral ("does-not-follow") recall** on the three
entailment families — the hardest class, where frontier models most often over-commit.

| Model | lsat_lr | logiqa | arct | folio (neu) | logicnli (neu) | proverqa (neu) | **mean** |
|---|--:|--:|--:|--:|--:|--:|--:|
| Base 4B (untuned) | 81.7 | 66.7 | 75.0 | 61.7 (50.0) | 48.3 (26.7) | 50.0 (63.3) | 63.9 |
| **LogicSLM (v7, 4B)** | 83.3 | 75.0 | 90.0 | 76.7 (83.3) | 90.0 (96.7) | 95.0 (96.7) | 85.0 |
| Claude Sonnet 4.6 | 94.9 | 85.0 | 88.3 | 80.0 (76.7) | 60.0 (26.7) | 69.5 (73.3) | 79.6 |
| Claude Opus 4.8 | 98.3 | 83.3 | 90.0 | 78.3 (80.0) | 93.3 (90.0) | 80.0 (90.0) | 87.2 |

The 4B model lands between Sonnet and Opus overall, and **beats both on ProverQA** and on the neutral
"does-not-follow" class.

### Robustness (LGMT metamorphic testing, n=300)

Answer stability under logic-preserving edits — reorder/duplicate premises, rewrite the conclusion,
rename symbols, rewrite formulas. **MVR = label-flip rate; lower is more robust.**

| Model | Acc | **MVR** | HDR | MR-P | MR-C | MR-S | MR-E |
|---|--:|--:|--:|--:|--:|--:|--:|
| **LogicSLM (v7)** | 87.5 | 9.3 | 5.3 | 1.2 | 7.8 | 17.9 | 13.4 |
| Claude Opus 4.8 | 85.0 | 13.9 | 9.1 | 12.9 | 20.3 | 7.9 | 12.8 |
| Claude Sonnet 4.6 | 76.9 | 9.7 | 2.8 | 9.8 | 9.7 | 10.5 | 9.4 |

LogicSLM is the most accurate **and** (with Sonnet) the most robust — it flips less than Opus overall
and far less on conclusion rewrites (MR-C **7.8** vs 20.3). Full corpus + eval details:
[`data-synthesis.md`](data-synthesis.md).

## Run the demo

A landing page with a live "compare LogicSLM to Opus and Sonnet" tool.

**Live:** deployed on Firebase Hosting at **https://logicslm.web.app** (project `logicslm`;
config in [`frontend/firebase.json`](frontend/firebase.json)). Redeploy with
`cd frontend && npm run build && firebase deploy`.

**Frontend** ([`frontend/`](frontend/)):
```bash
cd frontend
npm install
npm run dev        # http://localhost:5173
```
Then paste the server URL (below) into the page's "Server URL" field and Connect.

**Server** ([`server/`](server/)) — runs on Google Colab (the 4B needs the GPU). Open
[`server/logicslm_server.ipynb`](server/logicslm_server.ipynb) on a **GPU** runtime and run all cells;
the last cell prints a public URL. It needs a tunnel:
- **ngrok** — make a free account at [ngrok.com](https://ngrok.com), copy your **authtoken** from the
  dashboard, put `NGROK_AUTH_TOKEN=<token>` in the Drive `.env` (or Colab Secrets), and set
  `TUNNEL="ngrok"` in the config cell.
- Or leave the default `TUNNEL="cloudflare"` (no account needed).

The server also proxies Opus/Sonnet via the gateway using `.env` secrets, so the browser never sees a
key. Details: [`server/README.md`](server/README.md).

## Where things live

- **Training data:** [`data/sft_train_v7.jsonl`](data/sft_train_v7.jsonl) — the file the trainer reads.
- **Held-out eval:** [`data/eval_items_clean.json`](data/eval_items_clean.json).
- **All corpus artifacts:** [`data/`](data/) (raw sources under `data/_raw/`, rebuild-only).
- **Model (Hugging Face):** [`VS-HuggingFace-Cloud/LogicSLM-v7`](https://huggingface.co/VS-HuggingFace-Cloud/LogicSLM-v7)
  — the LoRA adapter (load on top of `Qwen/Qwen3.5-4B`).
- **Adapter in this repo:** [`runs/colab/adapter/`](runs/colab/adapter/) — the same LoRA weights +
  tokenizer locally (`adapter_model.safetensors`, `adapter_config.json`, …). Note: `runs/` is gitignored,
  so this exists only after training/download; it is regenerated by a run.
- **Where deployed adapter should be stored (Drive):** `/content/drive/MyDrive/SLM/runs/colab/adapter` (see
  `colab_standalone.py` / the server notebook).
